Introduction

Chime has a C-like syntax, and it's basic features are easy to pick up.  This tutorial covers only a few of the features, but should be enough to get you started.  It is designed to take about 15 minutes to complete.

Installation

You can download and install Chime by selecting one of the download options to the right.  Once you have Chime installed, you're ready to begin writing code.

hello, world

To write our first program, start by creating a new plain-text document.  Enter the following, and save it as Tutorial.chm.

"hello, world".print()

"hello, world" is a string object, and print() is a method called on that object.  print writes a string to stdout.

Compile and Execute

By default, Chime will automatically compile and excute your program.  We can run our new program by opening a terminal, moving to the directory we've saved Tutorial.chm to, and executing:

# chime Tutorial.chm
hello, world

And there we go!  If you take a look in the current directory, you'll see that chime has produces two additional files.  "tutorial.chc" is the object-file version of your program - compiled but unlinked.  "tutorial" is the actual binary, linked to both the chime runtime and chime core library and is a standalone executable.

Expressions, Loops, and Closures

This next step will combine three concepts into one.  We'll see how chime evaluates expressions and how closures work.  We can try this out by modifying Tutorial.chm to the following:

(2*2).times() do
{
    "hello, world".print()
}

 Execute your program by running chime again.

There's a lot going on here, so let's take it one part at a time.  First, we're doing a little math in the parenthesis - resulting in the integer 4.  Next, we are invoking the method times on that integer.  The times method executes a provided block of code the number of times represented by its object value.  Finally, we have the do statement.  do creates a new closure object, capturing whatever is in braces.  Adding do to the end of a method call is equivalent to making it the last argument of the call.  We'll see how that works next.

Onwards!
