Classes

Chime is an object-oriented language.  You can, of course, define your own classes using the implementation keyword.

implementation Greeter
{
    method greet()
    {
        "hello, world".print()
    }
}

greeter = Greeter.new()
greeter.greet()

Everyone loves inheritance and Chime supports single-inheritance like this:

implementation Greeter
{
    method greet()
    {
        "{self.greeting()}, world".print()
    }
    method greeting()
    {
        return "hello"
    }
}

implementation FormalGreeter : Greeter
{
    method greeting()
    {
        return "salutations"
    }
}

greeter = FormalGreeter.new()
greeter.greet()

Traits

You might be more familar with the concept of Roles in Perl 6, Modules in Ruby.  Traits are mixins that let you define behavior without using inheritance.

trait Greetable
{
    method greet()
    {
        "hello, world".print()
    }
}

implementation Greeter
{
    include Greetable
}

greeter = Greeter.new()
greeter.greet()

Traits are not classes - they cannot be subclassed or instantiated.

Built-In Classes

Chime features a few classes built into the runtime.  Here are a few that can be created with literal forms:

// Ranges
"hello, world"[0:6].print()
"hello, world"[7:-1].print()

// Arrays
["hello", ",", "world"].each() do (string)
{
    string.print()
}

// Hashes
hash = {"a":"world", "b":"hello", "c":","}
hash["b"].print()
hash["c"].print()
hash["a"].print()
