Closures and Arguments

 Check out this next program:

4.times( do { "hello, world".print() } )

Definitely more dense, this illustrates how a tailing-do is really the same as an inline do.  But, just to further re-inforce the issue, consider this:

function = do { "hello, world".print() }
4.times(function)

Here, we've created a variable to hold our closure and passed it in as the argument to "times".

Methods

Chime has a few different concepts for methods.  The simplest looks like this:

method hello()
{
    "hello, world".print()
}

self.hello()

In this case, self refers to the object instance in the current scope.  We're at the very top level of a file, which Chime calls the "Module Scope".  "self", in this case, refers to the loadable module this file defines.

Classes

Chime is an object-oriented language.  You can, of course, define your own classes using the "implementation" keyword.

implementation Greeter
{
    method greet
    {
        "hello, world".print()
    }
}

greeter = Greeter.new()
greeter.greet()

Everyone loves inheritance and Chime supports single-inheritance like this:

implementation Greeter
{
    method greet
    {
        "hello, world".print()
    }
}

implementation UniversalGreeter : Greeter
{
    method greet
    {
        "hello, universe".print()
    }
}

greeter = UniversalGreeter.new()
greeter.greet()

Traits

You might be more familar with the concept of Roles in Perl 6, Modules in Ruby.  Traits are mixins that let you define behavior without using inheritance.

trait Greetable
{
    method greet
    {
        "hello, world".print()
    }
}

implementation Greeter
{
    include Greetable
}

greeter = Greeter.new()
greeter.greet()

Traits are not classes - they cannot be subclassed or instantiated.
